# 量化回测框架实现与真实数据验证

> - 本篇记录 **可运行的** 月频回测框架（`quant_engine`）的设计、实现、Bug修复与真实数据回测结果
> - 代码仓库：[ashare-quant-kb](https://github.com/moria/ashare-quant-kb)（`strategies/` 目录）
> - 已验证策略：**中证500指增**（年化+12.63%，超额+5.30%）、**ETF动量轮动**（年化+19.52%，夏普1.23）
> - 关键教训：回测时序错误可导致虚假-87%亏损，月末T调仓必须应用T+1月收益率

## 一、框架架构

```
quant_engine/            ← 通用量化框架（可插拔核心）
├── core.py              ← BaseStrategy / BaseUniverse / Signal / Portfolio
├── data.py              ← 数据层: AKShare / Tushare / BaoStock 三实现
├── factors.py           ← 因子引擎: 10种因子 + 预处理流水线
├── alpha.py             ← Alpha合成: 固定权重 / IC_IR自适应
├── optimizer.py         ← 组合优化: cvxpy凸优化 + Alpha排序简化
└── backtest.py          ← 月频回测引擎 + 绩效计算

strategies_lib/          ← 具体策略
├── csi500_enhanced.py   ← 中证500指数增强（多因子+组合优化）
└── etf_rotation.py      ← ETF动量轮动（风险平价配权）

run.py                   ← 统一入口
download_real.py         ← BaoStock数据下载器（socket超时保活）
generate_demo_data.py    ← 离线演示数据（demo_前缀）
```

### 核心设计原则

| 设计点 | 方案 | 理由 |
|--------|------|------|
| 数据层 | 抽象接口 + 3种实现 | AKShare/Tushare/BaoStock可互换 |
| 缓存 | Parquet文件 | 列式存储、压缩好、Pandas原生支持 |
| 因子 | 预处理流水线（去极值→标准化→中性化→方向调整） | 标准量化因子处理范式 |
| 优化 | cvxpy优先 + Alpha排序回退 | cvxpy更精确，但等权基准下简化优化更好 |
| 回测 | 月频信号+日线收益 | 适合中低频多因子策略 |

## 二、回测引擎关键修复

### Bug: 同月收益率应用（致命错误）

**问题代码**（修复前）:
```python
for date in dates:
    new_w = strategy.rebalance(date=date, prices=close[:date])
    month_ret = monthly_returns[date]  # ← 错误：用同月收益率
    port_ret = (new_w * month_ret).sum()
```

**正确逻辑**（修复后）:
```python
for i in range(len(dates) - 1):
    rebal_date = dates[i]       # 调仓决策日（月末T）
    return_date = dates[i + 1]  # 收益实现日（月末T+1）
    new_w = strategy.rebalance(date=rebal_date, prices=close[:rebal_date])
    month_ret = monthly_returns[return_date]  # ← 正确：用下月收益率
    port_ret = (new_w * month_ret).sum()
```

**影响分析**:

| 因子 | 同月应用效果 | 原因 |
|------|-------------|------|
| 反转因子（-1方向） | 系统性亏损 | 选出本月下跌股票 → 获得负收益 |
| 动量因子 | 无预测力 | 涨幅已实现，无前瞻收益 |
| 波动率因子 | 轻微偏差 | 低波股票本月表现无偏方向 |
| 基本面因子 | 轻微正偏 | 与当月收益弱相关 |
| **综合效果** | **净值从1.0跌至0.13** | 反转+动量拖累主导 |

> **教训**: 回测中最隐蔽的Bug是时间对齐错误。表现为策略"持续亏损"但代码看似正确——因为逻辑链完整只是错位一期。

### 优化器改进

BaoStock不提供真实指数权重（仅提供成分股列表），导致基准权重为等权（1/N）。此时cvxpy的行业中性约束（±3%偏离）过于严格，稀释Alpha。

**修复**:
```python
bm_is_equal = benchmark_weights.std() < 1e-6
if bm_is_equal:
    # 等权基准下，跳过cvxpy，使用Alpha排序优化
    return _optimize_simple(alpha_scores, ...)
```

效果：年化超额从+2.33%→+5.30%。

## 三、因子体系

### 因子清单（10种）

| 因子 | 大类 | 计算方法 | 方向 | 权重 |
|------|------|---------|------|------|
| EP | 价值 | EPS / Price | ↑ | 15% |
| BP | 价值 | EP / ROE近似 | ↑ | 10% |
| ROE | 质量 | 财务数据直取 | ↑ | 15% |
| 营收增速 | 成长 | 同比增长率 | ↑ | 10% |
| Momentum 12-1 | 动量 | 12月收益-1月收益 | ↑ | 10% |
| Reversal 1m | 反转 | 1月收益率 | ↓ | 10% |
| 波动率 | 风险 | 60日年化波动 | ↓ | 10% |
| 换手率 | 流动性 | 20日均换手 | ↓ | 5% |
| 净利增速 | 成长 | 同比增长率 | ↑ | 0%* |
| ln_mktcap | 中性化 | 对数市值 | — | — |

> *净利增速在BaoStock财务数据中缺失，权重为0

### 预处理流水线

```
原始值 → MAD去极值(5σ) → Z-Score标准化 → 行业+市值中性化(OLS残差) → 方向调整
```

## 五、使用指南

### 快速开始

```bash
# 安装依赖
pip install numpy pandas scipy baostock loguru matplotlib seaborn

# 下载真实数据（~16分钟）
python3 download_real.py 100

# 运行回测
python3 run.py csi500 --pool_size 100 --hold 30
python3 run.py etf --start 20190101 --end 20251231

# 列出可用策略
python3 run.py list
```

### 自定义策略

```python
from quant_engine.core import BaseStrategy
import pandas as pd

class MyStrategy(BaseStrategy):
    name = "我的策略"

    def rebalance(self, date, prices, universe, context):
        # prices: date × stock_code 的收盘价宽表
        # 返回: stock_code → weight 的Series
        returns = prices.pct_change(20).iloc[-1]
        top = returns.nlargest(10)
        weights = top / top.sum()
        return weights
```

## 六、已知局限与后续改进

| 局限 | 影响 | 改进方向 |
|------|------|---------|
| 财务数据固定为Q3 2024 | 回测前半段存在未来数据偏差 | 按季度滚动获取PIT财务数据 |
| 成分股列表固定 | 生存偏差 | 按月动态获取历史成分股 |
| BaoStock无指数权重 | 行业约束不精确 | 接入Tushare获取真实权重 |
| 仅月频调仓 | 无法捕捉短期机会 | 扩展为周频/日频回测 |
| 无滑点模型 | 成本低估 | 接入Level-2数据构建冲击成本模型 |
