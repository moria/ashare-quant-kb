### 工具选择原则

**根据主题特点选择最合适的方法：**

| 场景 | 推荐策略 |
|------|----------|
| 需要从本地 PDF 提取规格参数 | `read_pdf.py --search-any` → 找到页面 → `--pages X-Y --tables --markdown` |
| 需要对比多个来源的数据 | 搜索多来源 → 写 Python 脚本整合 |
| 找到了有价值的外部 PDF | `curl` 下载 → `read_pdf.py` 提取关键信息 |
| 需要尺寸示意图 | 创建 Mermaid/SVG 示意图 |
| 需要大量数据汇总 | 写脚本聚合 → 生成 markdown 表格 |
| 搜索结果质量差 | 换关键词、换语言、直接去官方来源 |
| 需要计算 | 写 Python 脚本 → 生成参数对照表 |

**主动使用脚本处理数据，不要手动搬运。**

## 调研流程

### 阶段一：策略规划

拿到主题后，先花 30 秒判断：
- 本地 PDF 中哪些章节相关？（查看 `pdf-sources.json` 的章节映射）
- 应该用什么关键词搜索 PDF？（查看 `keyword_hints` 翻译表）
- 是否需要写脚本处理提取的数据？
- 需要哪些类型的示意图？

### 阶段 1.5：本地 PDF 提取（如有 PDF 可用）

1. **读取 PDF 注册表**：读取 `pdf-sources.json`，了解可用的本地 PDF（章节、关键词映射）。
2. **关键词映射**：从调研主题提取关键词，用 `keyword_hints` 翻译。
3. **定位目标章节**：用 `chapter_map` 找到相关页码范围。
4. **摘要搜索**：执行多关键词摘要搜索，快速定位高价值页面：
   ```bash
   python3 read_pdf.py --file "路径/PDF文件" \
     --search-any "关键词1,关键词2,关键词3" \
     --pages <章节范围> --summary --max-results 10
   ```
5. **精确提取**：发现高价值页后，全文+表格模式精确提取：
   ```bash
   python3 read_pdf.py --file "路径/PDF文件" \
     --pages X-Y --tables --markdown
   ```
6. **建立数据骨架**：将提取数据整理为笔记的骨架。
7. **记录缺口**：标注 PDF 未覆盖的内容，留待网络调研。

### 阶段二：网络调研（如 project.json 中已启用）

用 WebSearch 和 WebFetch 补充 PDF 未覆盖的空白：
- 搜索标准、法规、最佳实践
- 获取官方文档和技术规格
- 交叉验证多个来源的准确性

如果网络搜索未启用，标注缺口为"本地资料未覆盖，待后续网络调研补充"。

### 阶段三：整合输出

将 PDF 数据和网络调研整合为结构化 Obsidian 笔记。
